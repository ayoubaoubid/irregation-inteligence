À Propos du Projet

Tisser le lien entre le génie logiciel, la data science et la viabilité agricole.

1. Motivation de l'Application

Cette application web démontre concrètement l'utilisation de frameworks backend robustes (Django) combinés aux avancées majeures du Machine Learning pour soulager des problématiques sociétales urgentes. Nous fournissons ici un prototype complet à la pointe de la technologie permettant de visualiser et d'inférer dynamiquement les besoins réels en irrigation sur toute sorte d'écosystèmes.

2. L'Architecture

🧑‍💻

Modèle d'Intelligence Artificielle de Régression & Déploiement Full-Stack MLOps

Ce projet illustre une fusion des interfaces UX responsives avec des serveurs de modélisation asynchrones avancés sous Python.

3. Pile Technologique

  • 🐍 Backend Logic : Python 3.12, Django MVT Framework
  • 🎨 Frontend UX : HTML5 Hybride Sémiotique, CSS3, JS Vanilla (DOM pur)
  • 📊 Dataviz Interactive : Lib. Chart.js (Rendus Canvas accélération)
  • 🧠 Calcul ML Analytique : Arbres de Régression via la librairie Scikit-Learn
  • 🗄️ Couche ORM/SGBD : SQLite structuré (Dev) / Déploiement PostgreSQL (Prod)

4. Paradigme Machine Learning

Notre matrice recourt à un entraînement dit "Supervisé" consistant à corréler une infinité de paramètres physico-chimiques du champ avec le rendement liquide requis. Ces paramètres de variance élevée (pH et micro-climat locaux) deviennent ainsi la matière première permettant de prédire efficacement le volume hydreux via un point d'entrée API rapide et léger.

5. Concept de Distribution DevOps

Imaginée de concert avec de bonnes pratiques MLOps modernes, la web-application s'abrite facilement sous le modèle des conteneurs Linux isolés (Docker Containers). Toutes ressources statiques sont isolées pour minimiser les charges de la requête initiale et fluidifier le serveur asynchrone final (type reverse-proxy Gunicorn vers Nginx).
